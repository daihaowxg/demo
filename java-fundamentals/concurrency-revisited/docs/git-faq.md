# Git 常见问题与技巧

## 1. .gitkeep 文件的作用

`.gitkeep` 文件并不是 Git 的核心功能或官方标准，而是一个**社区约定俗成**的习惯用法。

它的主要作用是：**强行让 Git 提交并保留一个空目录**。

### 核心原因
**Git 是不追踪空目录的**。
如果你在项目中创建了一个空的文件夹（例如 `logs/` 或 `uploads/`），但里面没有任何文件，当你运行 `git add .` 时，Git 会直接忽略这个文件夹。当你把代码推送到远程仓库，并在另一台机器 `pull` 下来时，这个文件夹就会丢失。

### 解决方案
为了在版本控制中保留这个目录结构，开发者通常会在这个空目录下放一个文件。
虽然你可以放任何文件（比如 `readme.txt`），但为了表明"**我只是想占个坑，保留这个文件夹，别无他意**"，大家约定使用名为 `.gitkeep` 的空文件。

### 常见场景
1. **预留目录结构**：比如标准的 Maven/Gradle 项目结构中，某些目录（如 `src/test/java`）初始可能是空的，但需要保留。
2. **临时文件存放地**：比如创建一个 `tmp/` 或 `logs/` 目录，并在里面放一个 `.gitkeep`，同时在 `.gitignore` 中忽略该目录下的其他文件（`*`），这样既保留了目录，又不提交实际的日志文件。
